Topological structure (Graph)

Formalizing the Difference

Traditional ML (e.g., MLP):

\[ \boldsymbol{h}^{(k)} = \varphi^{(k)}\left( \boldsymbol{h}^{(k-1)}\boldsymbol{W}^{(k)} + \boldsymbol{b}^{(k)}\right) \]

Independent row-wise processing.

Message Passing (GNN):

\[ \mathbf{h}_v^{(k)} = \phi^{(k)}\!\left( \mathbf{h}_v^{(k-1)}, \bigoplus_{u \in \mathcal{N}(v)} \psi^{(k)}\!\left(\mathbf{h}_v^{(k-1)}, \mathbf{h}_u^{(k-1)}, \mathbf{e}_{vu}\right) \right) \]

Captures the neighborhood \(\mathcal{N}(v)\) and edge features \(\mathbf{e}_{vu}\).

Neighborhood Explosion

  • A node's output depends on its K-hop neighborhood.
  • For many real-world graphs this neighborhood grows too large to fit on the training machine.

Solution: Sampling

Explored strategies
  • GraphSAINT: Subgraph sampling to decouple GNN depth from neighborhood size.
  • GraphSAGE: Neighborhood sampling (fixed-size).
    • Uniformly samples $k$ neighbors per hop.
    • Allows mini-batch training on massive graphs.

GraphSAGE-sampling

Sample a subgraph around each seed node.

Fanout: \([3, 2]\)

Sampling Changes the Training Loop

for batch B of seeds:
    G_B      ← Sample(B)          # sampling
    X_B, E_B ← Fetch(G_B)         # feature retrieval
    ŷ        ← f_θ(X_B, E_B)      # forward
    L        ← CE(ŷ, y_B)         # loss
    θ        ← θ − η ∇L           # update

 

PyTorch Geometric (PyG)

  • Industry standard for GNNs.
  • RAM-based: Loads entire graph into memory.
  • Easy GPU usage.
  • The Cost: High memory requirement; scaling to billions of edges is very expensive.

Neo4j: The Graph Platform

  • Tabular vs Graph: Relationships are first-class citizens.
  • Cypher: Declarative graph query language.
  • UDP (User Defined Procedures): Custom Java-based extensions may be deployed as DB-plugins.

Our Work

Functional Equivalence

Validating that Neo4j samplers (UDP & Cypher) match PyG's native sampling.

  • KL Divergence: Measuring distribution similarity between samplers.
  • Convergence: Comparing Validation Accuracy vs. Epochs.

Pipeline Optimizations

  • UDP Samplers: Bypassing Cypher overhead for raw traversal speed.
  • Pre-aggregation: Moving computations into the database to reduce data transfer.

In-Database Inference

Exploring where to run the model:

  1. Cypher: Pure query-based inference.
  2. UDP: Java-based model execution.
  3. Hybrid: UDP for sampling → PyG for prediction.
  4. Baseline: Only PyG (full memory).

1. Cypher Inference

  • Weights: All model parameters passed in every call.
  • Hard-coded Architecture: Model layers baked directly into the query structure.
  • Two Flavors:
    • Normal: Standard arithmetic in Cypher.
    • Optimized: Uses Neo4j's native VECTOR operations.

Large, self-contained queries.

2. Java UDP Inference

Persistent Model State

  • spec.json: Defines the model layout.
  • weights.bin: Stores pre-trained parameters.
  • Lifecycle: Uploaded via dedicated UDP once; cached in-memory for all subsequent calls.
